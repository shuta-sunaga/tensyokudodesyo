# AI×転職 テーマ選定スキル

`docs/ai-knowhow-growth-strategy.md` の成長戦略に基づき、アクセス増を狙うノウハウ記事テーマを選定してください。テーマ確定後は `/generate-knowhow` のパイプライン（Stage 2以降）で記事を生成します。

**このスキルの出口はテーマ提案（ユーザー承認待ち）。承認後は /generate-knowhow の Stage 2〜9 を自動実行すること。**

---

## ユーザー補足・要望（任意）

`$ARGUMENTS` に補足が渡される場合がある。例: `/ai-knowhow-theme ピラー記事を作りたい` `/ai-knowhow-theme Gemini系で`

- **空の場合**: 下記の優先順位ロジックで自動提案
- **指定がある場合**: 指定を最優先で反映しつつ、戦略との整合（鮮度・カニバリ・ピラー構造）は必ずチェックする

---

## Step 1: データ収集（並行実行）

1. **戦略ドキュメント**: `docs/ai-knowhow-growth-strategy.md` を読む（KPI・ロードマップ・モデル状況表）
2. **既存記事**: `public_html/data/knowhow.json` + `public_html/data/knowhow-mt.json` の全タイトル・カテゴリ
3. **GSC実績**: `reports/` の最新月次レポート（`gsc-YYYY-MM.md`）から上位ページ・伸びたページ・落ちたページ
4. **最新モデル動向**: WebSearch で以下を確認（**戦略ドキュメントのモデル表は古い可能性があるため必ず検索する**）:
   - `OpenAI GPT latest model {当月}` 
   - `Google Gemini latest model {当月}`
   - `Anthropic Claude latest model {当月}`
   - 直近30日以内にリリースされた新モデルがあれば **最優先候補（First-Mover枠）** とする

## Step 2: カバレッジマトリクス分析

「モデル/ツール × 転職フェーズ」のマトリクスを作り、空白セルを特定する:

- **行**: ChatGPT（+ GPT-x.x モデル名）/ Claude（+ Fable/Opus/Sonnet）/ Gemini（+ 3.x）/ Copilot / Perplexity / NotebookLM / その他新興ツール
- **列**: 自己分析 / 職務経歴書・履歴書 / 面接対策 / 企業・業界研究 / 退職・入社・年収交渉
- 既存記事をセルに配置し、空白＝候補。**GSCで実績のある列（自己分析 > 職務経歴書 > 面接対策）の空白を優先**

## Step 3: 優先順位ロジック

以下の順でテーマ候補をスコアリングする:

| 優先度 | 枠 | 条件 |
|-------|-----|------|
| 1 | 🔥 First-Mover | 直近30日以内リリースの新モデル × 勝ちフェーズ（未執筆） |
| 2 | 📈 勝ちパターン横展開 | GSC実績上位テーマの別モデル版（未執筆） |
| 3 | 🏛 ピラー新設・更新 | モデル比較・完全ガイド系。ピラー不在 or 大型リリース後に比較表が陳腐化している場合 |
| 4 | 🧩 マトリクス空白 | 実績中位フェーズの空白セル |
| 5 | ♻️ リライト | 月次レポート「落ちたページ」のモデル名記事の鮮度更新 |

## Step 4: テーマ提案（⏸ ユーザー承認で停止）

3件提案し、以下のテーブルで提示して**停止**する:

```
| # | 枠 | カテゴリ | タイトル案（32字以内） | 想定検索KW | カニバリチェック | 根拠（GSC/鮮度） |
```

- **カニバリチェックは全既存記事タイトルと突合**。同一「モデル×フェーズ」記事が既にある場合は提案しない
- 新記事が既存エース記事（例: knowhow-058）と近接する場合は、検索意図の差別化方法と相互リンク方針を明記する
- ピラー記事を提案する場合は、リンクを張るクラスター記事のID一覧も添える
- おすすめ1件に「(推奨)」を付け、理由（鮮度・実績・KPI貢献）を一文で示す

## Step 5: 承認後の処理

1. 確定テーマで `/generate-knowhow` の Stage 2〜9 を実行（記事生成 → 画像 → 統合 → 検証 → git → デプロイ → GSC → X投稿）
2. **追加要件（このスキル経由の記事に必須）**:
   - 本文に**正式モデル名・バージョン表記**を使い、冒頭で「{YYYY年M月}時点の情報」であることを明記する
   - 関連記事リンクは戦略のピラー・クラスター構造に従う（クラスター記事→ピラー + 同フェーズの実績記事）
   - モデルの料金・提供プラン（無料/有料）に言及する場合はWebSearchで裏取りする
3. 大型モデルのリリース直後（First-Mover枠）で記事を作った場合は、ピラー記事（モデル比較）の比較表更新も提案する
4. 完了後、`docs/ai-knowhow-growth-strategy.md` のモデル状況表に「執筆済み」を反映する

---

## 注意事項

- カニバリチェック必須（feedback_knowhow_cannibalization 準拠）
- タイトル32文字以内 / モデル名は正式表記
- Stage 1（テーマ承認）以外は停止しない。bashコマンドの実行確認は不要
- KPI進捗（クリック数・058依存度）は毎月の GSC レポート時に確認し、乖離が大きい場合は戦略ドキュメントの改訂を提案する
