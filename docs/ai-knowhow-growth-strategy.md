策定日: 2026-07-16 / 対象: 転職ノウハウ（`/knowhow/`）のオーガニック流入拡大

`/ai-knowhow-theme` スキル（`.claude/commands/ai-knowhow-theme.md`）がこのドキュメントを参照してテーマ選定を行う。**戦略の変更はこのファイルを更新すること。**

3. **ハブ不在**: AI記事が60本超あるが、回遊させるピラー（まとめ・比較）構造が弱い

## 2. 戦略の3本柱

### 柱1: 勝ちパターンの横展開（Proven Topic × New Model）

実績のあるテーマ（自己分析 > 職務経歴書 > 面接対策）を、**新しいモデル名・バージョン名で再展開**する。

- 例: 「ChatGPT自己分析」(058) が勝ち → 「GPT-5.6で自己分析」「Claude Fable 5で自己分析」を出す
- 既存エース記事とはH1・検索意図を明確に分け、**相互リンクで束ねて共倒れを防ぐ**（新記事→エース記事へ関連記事リンク必須）

### 柱2: モデル名の鮮度戦略（First-Mover）

**新モデルのリリースから2週間以内に「{モデル名}×転職」記事を公開する。** モデル名KWはリリース直後が最も競合が薄く、指名検索が急増する。

- リリース検知: テーマ選定時に必ずWeb検索で最新モデル動向を確認（`/ai-knowhow-theme` に組込済み）
- 「何が変わったか・転職活動でどう使うか」を最速で解説するポジションを取る

### 柱3: ピラー・クラスター構造（回遊と網羅性）

```
[ピラー] AIモデル比較・完全ガイド（比較・使い分け・最新動向）
   ├─ knowhow-207 転職のAI活用完全ガイド（ツール名レベル・既存）
   ├─ [新設] 最新AIモデル比較（モデル名レベル・年次更新型）
   │
[クラスター] モデル名 × 転職フェーズ
   ├─ 自己分析: 058(ChatGPT) / 129(Claude) / 144(Gemini) / [GPT-5.6…]
   ├─ 職務経歴書: 125(Claude) / 176(Gemini) / 128(Copilot) / …
   ├─ 面接対策: 108・174・126・142 / …
   ├─ 企業研究: 118・127・109・160 / …
   └─ 退職・入社: 177・150・169 / …
```

- ピラーは「比較・どれを選ぶか」の検索意図、クラスターは「このモデルでどうやるか」の検索意図
- **クラスター記事は必ずピラーへ、ピラーは主要クラスターへ内部リンク**
- ピラーはモデルリリースのたびに本文更新（lastmod更新 + GSC再送信）してポジション維持

## 3. コンテンツ運用ルール

### 鮮度メンテナンス

- モデル名を含む記事は**四半期ごと**に記載モデルの最新性をレビュー（GSC月次レポートのタイミングで確認）
- 大型モデルリリース時: ①新記事を2週間以内に公開 ②ピラー記事の比較表を更新 ③関連クラスターの旧モデル記述に追記
- 順位が落ちた記事（月次レポート「落ちたページ」）はリライト候補に載せる

### カニバリ防止

- 同一「モデル×フェーズ」の重複記事は作らない（ツール名記事とモデル名記事は検索意図が違えば併存可）
- 新記事は既存の近接記事へ関連リンクを張り、H1とdescriptionで意図を差別化する

### タイトル規則

- 32文字以内 / モデル名は正式表記（GPT-5.6, Gemini 3.5 Flash, Claude Fable 5 等）
- 「{モデル名}で{転職アクション}｜{差別化要素}」を基本形とする
