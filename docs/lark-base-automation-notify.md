# Lark Base 自動通知の設定手順

転職相談フォームの送信があった際、**Lark の指定メンバーに複数同時で通知** を送るための設定手順です。

「**転職どうでしょう - お問い合わせ**」Base に Automation（ワークフロー）を1つ追加するだけで動きます。コード変更は不要です。

---

## 完成イメージ

新規送信があると、指定したメンバー全員に Lark のメッセージが届きます：

```
🔔 新しい転職相談が届きました

氏名: 山田 太郎
都道府県: 滋賀県
年齢: 32歳
気になった求人: 法人営業/未経験歓迎
ご状況: 在職中

▼ 詳細を見る (Lark Base のレコードを開く)
```

メンバーは Lark の通知 + DM/グループ で気づき、Base を開いて対応できます。

---

## Step 1: Base を開く

1. Lark にログイン
2. 「**転職どうでしょう - お問い合わせ**」Base を開く

## Step 2: 自動化メニューを開く

1. Base 画面右上の「**...** (More)」または上部メニューから「**Automation (自動化)**」をクリック
   - 場所が見つからない場合: 表示モード切替アイコン横の「⚡」マークを探す
2. 「**Add Workflow / 新規ワークフロー作成**」をクリック

## Step 3: トリガーを設定（レコード追加で発動）

1. 「**Trigger (トリガー)**」を選択
2. 「**When a record is created (レコードが追加されたとき)**」を選択
3. テーブルが「submissions」(または最初に作ったテーブル) になっていることを確認
4. 「Next」または「OK」

## Step 4: アクションを追加（通知送信）

1. 「**Add Action (アクション追加)**」をクリック
2. アクションリストから「**Send a Lark message (Larkメッセージを送る)**」を選択

### 4-1. 送信先（複数メンバー指定）

「**Recipients (送信先)**」欄で、通知を受け取りたいメンバーを **複数選択**：

- 個別ユーザー名をタイプして検索 → 選択（例: `shuta.sunaga@sei-san-sei.com` 等）
- 必要なメンバー全員を追加（5人でも10人でも追加可）
- グループチャットも指定可能（既存の運用チャット選択）

### 4-2. メッセージタイプ

「**Message type**」を選択：

- **Card message (カード)** — 推奨: 見やすい・リッチ
- **Plain text** — シンプル

#### 推奨: カードメッセージのテンプレート

タイトル例:
```
🔔 新しい転職相談が届きました
```

本文（Lark UI の field 挿入機能でフィールド変数を埋め込む）:
```
氏名: {{name}}
ふりがな: {{name_kana}}
都道府県: {{prefecture}}
年齢: {{age}}歳
メールアドレス: {{email}}
電話番号: {{phone}}
現在の状況: {{current_status}}
気になった求人: {{interested_job}}

ご質問・ご要望:
{{message}}

受信日時: {{submitted_at}}
```

※ `{{...}}` は Lark の UI 上で「フィールド挿入」ボタンから選ぶと自動でテンプレ化されます（手書きではなく挿入機能を使う）。

#### ボタン（任意・カードメッセージの場合）

「Add button」で「Base レコードを開く」ボタンを追加：
- ボタンテキスト: `詳細を見る`
- アクション: `Open record (このレコードを開く)`

### 4-3. 送信タイミング

「**Send immediately (即時送信)**」を選択（デフォルト）。

## Step 5: ワークフロー名を設定 → 保存

1. ワークフロー名（任意）: `新規お問い合わせ通知`
2. 「**Save / 保存**」 + 「**Enable / 有効化**」

## Step 6: テスト

1. ローカル開発環境のフォーム送信、またはテストレコード手動追加で動作確認
2. 指定メンバー全員にメッセージが届けば成功

---

## ハマりやすい点

| 症状 | 対処 |
|------|------|
| 「Automation」メニューが見当たらない | 旧UIの場合は「Settings → Workflows」、新UIなら「⚡」アイコン |
| 通知が来ない | ワークフローが「Enabled (有効)」になっているか確認 / トリガーが「レコード追加」か確認 |
| メッセージにフィールド値が入らない | フィールド挿入は手書きでなく、UIの「{}」マークから挿入する |
| 「Recipients」に検索しても出ない | 該当ユーザーが Lark テナントの正規メンバーか確認 |

---

## オプション機能

### A. 重要度フィルタ（条件付き通知）

例: 年齢 25-40歳 + 都道府県=滋賀県 のみ通知 → アクション前に「Condition」を追加。

### B. 担当者割当

「**Assign to (担当者)**」フィールドが status カラムにあれば、レコード追加時に round-robin で自動割当→担当者にだけ通知 等も可能（やや高度）。

### C. 営業時間外フィルタ

「**Wait until (時間条件)**」アクションを挟むと「翌営業日朝に送信」等の制御も可。

---

## このワークフローはいつでも UI で変更できる

- 通知メンバーの追加・削除
- メッセージ文面の調整
- 通知タイミング（即時 → 1時間ごとまとめ）
- 条件の追加（特定都道府県のみ等）

すべて Lark UI 上で完結します。コード変更不要。

---

設定完了したら、テストフォーム送信してメンバーに通知が届くか確認してください。届かない場合は、ワークフロー画面の「Run history (実行履歴)」でエラー内容を確認できます。
