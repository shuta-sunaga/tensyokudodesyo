# MTテンプレート反映手順（LINE→お問い合わせ CTA 置換）

LINE相談CTAを `/contact/` フォームに切り替えた変更を、Movable Type で生成されているページに反映するための手順です。

## 影響範囲

11ファイルのMTテンプレートが更新されています：

| ファイル | テンプレート種別 | MT管理画面のメニュー |
|---------|--------------|------------------|
| `mt-template/header-html.mtml` | テンプレートモジュール | デザイン → テンプレートモジュール |
| `mt-template/footer-html.mtml` | テンプレートモジュール | 同上 |
| `mt-template/index-html.mtml` | インデックステンプレート | デザイン → インデックステンプレート |
| `mt-template/interviews-html.mtml` | インデックステンプレート | 同上 |
| `mt-template/companies-html.mtml` | インデックステンプレート | 同上 |
| `mt-template/knowhow-html.mtml` | インデックステンプレート | 同上 |
| `mt-template/prefecture-page.mtml` | アーカイブテンプレート | デザイン → アーカイブテンプレート |
| `mt-template/job-detail-child.mtml` | アーカイブテンプレート | 同上 |
| `mt-template/interview-detail.mtml` | アーカイブテンプレート | 同上 |
| `mt-template/company-detail.mtml` | アーカイブテンプレート | 同上 |
| `mt-template/knowhow-detail.mtml` | アーカイブテンプレート | 同上 |

## 反映手順（テンプレート毎）

### 1. MT管理画面にログイン
通常運用しているMT管理URLにアクセス。

### 2. 該当ブログ（「転職どうでしょう」）を選択

### 3. 各テンプレートを開いて中身を上書き

各 `.mtml` ファイルを開き、内容をすべて選択 → コピー
MT管理画面の対応するテンプレート編集画面で、本文を全選択 → 削除 → ペースト

例：
- `mt-template/header-html.mtml` → MT「テンプレートモジュール」内の **header_html**（または同等の名前）を開いて上書き

### 4. 「保存」 + 「再構築」

各テンプレートを保存すると「保存して再構築」または「保存」ボタンがあります。**再構築まで実行**してください。

特にインデックステンプレートとアーカイブテンプレートは「再構築」必須です。

### 5. 一括再構築（推奨）

すべてのテンプレートを保存後、**「ツール」 →「再構築」 →「すべての項目」** で全体再構築をかけると、次のページ群が `/contact/` リンク版に更新されます：

- トップページ `/index.html`
- 一覧ページ `/interviews/`, `/companies/`, `/knowhow/`
- 都道府県ページ `/shiga/`, `/shizuoka/`
- 求人詳細ページ `/{prefecture}/jobs/*.html`
- インタビュー詳細 `/interviews/detail/*.html`
- 企業詳細 `/companies/detail/*.html`
- ノウハウ詳細（MT生成分） `/knowhow/detail/{id}.html`
- ヘッダー/フッター `/includes/header.html`, `footer.html`

## 確認方法

再構築後、本番ブラウザで以下を確認：

1. ヘッダー右上の「LINEで相談」ボタンが「**お問い合わせ**」に変わっているか
2. フッターのリンクに「**利用規約**」「**お問い合わせ**」が並んでいるか
3. トップページの大きな緑色CTAが「**お問い合わせフォームへ**」になっているか
4. `/interviews/`, `/companies/`, `/knowhow/` の各一覧ページのCTAが緑グラデの「お問い合わせはこちら」になっているか
5. 求人詳細・インタビュー詳細等の各種CTAも同様に変わっているか

## 注意

- **再構築前に1テンプレートだけ更新するとサイトが不整合になる可能性あり**。`header_html` を更新したら全ページに影響するので、すべてのテンプレート更新→全体再構築を一気に実施するのが安全。
- 再構築前にバックアップを取っておくと安心（MT管理画面の「エクスポート」または該当ページのHTMLをDLしておく）。
- もし`/includes/header.html`が私のSCPデプロイで上書きされていた場合（本来MT管理ですが、deploy.shのブロックリスト前にデプロイされた可能性）、再構築すれば MT が正しい内容で上書きします。

## 何かおかしい時

- 「LINEで相談」が一部ページに残る → そのページの**個別アーカイブテンプレート**をMT管理画面で再保存→再構築
- 個別記事の本文中に古いLINEリンクがある → 本文編集が必要（テンプレートではなくコンテンツ側）
